# Code Review Skill

Review code for quality, security, and best practices.

## When to Use

- Reviewing pull requests
- Debugging complex issues
- Auditing code quality
- Security assessments

## Process

1. **Understand the context**: Read surrounding code and comments
2. **Check for bugs**: Look for obvious errors and edge cases
3. **Verify error handling**: Ensure all error paths are covered
4. **Review security**: Check for vulnerabilities (injection, XSS, etc.)
5. **Assess readability**: Is the code self-documenting?

## Key Questions

- Does the code handle edge cases?
- Are there any security concerns?
- Is the code readable and maintainable?
- Are there adequate tests?

## Best Practices

- Use the `check_security` tool for automated vulnerability scanning
- Reference the checklist for comprehensive reviews
- Document any concerns or suggestions
